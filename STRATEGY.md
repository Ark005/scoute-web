# Scout·E Web — Стратегия миграции Next.js

## Текущее состояние (12 апреля 2026)

### Защита от скачивания (ГОТОВО ✅)

**3 уровня защиты:**

| Уровень | Что сделано |
|---------|-------------|
| **Nginx** | Блокировка скрейперов (Scrapy, HTTrack, WebCopier) по User-Agent. Security headers (nosniff, SAMEORIGIN, noindex для API). Кэширование _next/ статики. |
| **Django API** | Rate limiting: 60 req/min для анонимов. CORS только scoute.app + localhost. Anti-scraping middleware: блокирует python-requests, wget, пустой UA без referer. API docs (/api/docs/) закрыты в продакшне. /admin/ доступен только с сервера. |
| **Headers** | X-Robots-Tag: noindex для API. Cache-Control: private, no-store. X-Frame-Options: DENY для API. |

**Результат проверки:**
- `curl + referer scoute.app` → 200 ✅
- `python-requests без referer` → 403 🚫
- `без User-Agent` → 403 🚫
- `/api/docs/` → 404 🚫
- `Googlebot/YandexBot` → 200 ✅ (SEO пропускаем)

## Распределение: Opus vs Sonnet

### Sonnet может делать самостоятельно:
- Страницы без сложной логики (визард, формы, списки)
- UI-компоненты (навигация, карточки, фильтры, скелетоны)
- SEO-задачи (meta, sitemap, structured data, robots)
- Стандартные интеграции (Web Share API, localStorage)
- Стили и адаптивность

### Opus делает сам:
- Интеграция с оплатой и session management
- AI-чат (streaming Claude responses)
- AI-генератор маршрутов (сложная форма + обработка ответа)
- Девопс (nginx, pm2, деплой, SSL)
- Архитектурные решения и дебаг
- PWA/Service Worker
- Аудиогид (Web Audio API)
- Performance оптимизация
